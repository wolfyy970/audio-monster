# Code of Conduct

## Our commitment

Audio Monster contributors, maintainers, and users are expected to make project
spaces welcoming, useful, and safe for everyone, regardless of background,
identity, experience, or ability.

## Expected behavior

- Be respectful, patient, and specific.
- Critique ideas and code, not people.
- Assume good intent while taking responsibility for impact.
- Accept constructive feedback and correct mistakes openly.
- Protect private information shared during debugging or security reporting.

Harassment, discrimination, threats, sexualized attention, deliberate
intimidation, personal attacks, and publishing another person's private
information are not acceptable.

## Scope

This applies in repository discussions, issues, reviews, project chat, events,
and public interactions where someone is representing Audio Monster.

## Enforcement

Report conduct concerns privately to a project maintainer through an available
private GitHub channel. Do not post sensitive evidence or personal information
in a public issue. Maintainers may edit or remove contributions, issue warnings,
temporarily restrict participation, or permanently remove participants when
necessary to protect the community.

Enforcement decisions should be prompt, proportionate, documented privately,
and free from conflicts of interest. A maintainer named in a report must not
adjudicate it.
