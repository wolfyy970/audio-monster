# Security policy

## Supported versions

Audio Monster is currently an early preview. Security fixes are made on the
latest revision of the default branch; older builds are not supported.

## Reporting a vulnerability

Please do not disclose a suspected vulnerability in a public issue. Use the
repository's **Security → Report a vulnerability** flow on GitHub. Private
vulnerability reporting is enabled. Include the affected revision, impact,
reproduction steps, and any suggested mitigation.

Reports about unsafe URL handling, WebKit extraction, filesystem access,
security-scoped bookmarks, iCloud placement, metadata leakage, model downloads,
or signing are especially useful.
